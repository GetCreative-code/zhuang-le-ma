# 「装了吗」项目提示词 — 完整解析

## 一、项目名称含义

**「装了吗」** 是一个双关语：

- 字面意思：**"装" = 安装**，"你装了吗？" = "你安装了吗？"
- 谐音梗：中文口语中 "装" 也有 "假装/装逼" 的意思，这是一个轻松搞笑的梗，符合产品调性 "轻松搞笑" 的定位
- 口号 **「今天你装了吗？」** 模仿了 "今天你吃了吗" 的问候句式

---

## 二、角色设定

> "你是一个全栈工程师，擅长 Python + Electron + React 开发。"

这是对 AI 的角色定位指令。它告诉 AI 扮演一个**全栈工程师**的角色（即同时能写前端和后端的工程师），并指定了三个核心技术方向。

---

## 三、架构总览（三个子系统）

整个项目分为三个子系统，构成 **前端桌面客户端 → 后端服务 → Web管理后台** 的三角架构：

| 子系统 | 面向用户 | 说明 |
|--------|----------|------|
| **Electron 桌面 APP** | 普通学生用户 | 安装在用户电脑上，提供安装软件的操作界面 |
| **Python 服务端** | APP和管理后台的"大脑" | 处理业务逻辑、AI调用、数据存储、提供API |
| **Web 管理后台** | 管理员 | 浏览器打开的网页，管理所有方案和反馈 |

## 四、专业名词详解

### 1. Electron [底层框架]

- **定义**：由 GitHub 开源的一个框架，可以用 Web 技术（HTML/CSS/JS）来构建跨平台的桌面应用程序
- **通俗理解**：把一个网页包装成桌面软件（像微信桌面版、VS Code、Discord 都是 Electron 做的）
- **关键特点**：一套代码同时运行在 Windows 和 macOS 上

### 2. React [前端框架]

- **定义**：由 Meta（Facebook）开源的前端 UI 框架，用于构建用户界面
- **通俗理解**：一个写网页界面的工具库，把页面拆成一个个可复用的"组件"
- **关键特点**：组件化开发、虚拟DOM、生态丰富

### 3. Shadcn UI [UI组件库]

- **定义**：一个基于 Radix UI 和 Tailwind CSS 构建的现代 UI 组件集合
- **通俗理解**：提供开箱即用的漂亮按钮、对话框、表单等组件，开发者直接复制代码到项目里用，不需要 npm 安装
- **关键特点**：代码完全可控、视觉精美、可定制性强

### 4. FastAPI [后端框架]

- **定义**：一个现代、高性能的 Python Web 框架，用于构建 RESTful API
- **通俗理解**：用 Python 写后端接口最快的方式之一，自带自动生成的 API 文档
- **关键特点**：异步支持、自动数据校验、速度极快（基于 Starlette 和 Pydantic）

### 5. SQLite [数据库]

- **定义**：一个轻量级的嵌入式关系数据库
- **通俗理解**：数据存在一个 `.db` 文件里，不需要安装数据库服务，适合小规模应用
- **关键特点**：零配置、无需服务进程、文件即数据库

### 6. RESTful API [接口规范]

- **定义**：REST（Representational State Transfer）风格的 Web API 设计规范
- **通俗理解**：一种用 HTTP 请求（GET/POST/PUT/DELETE）来增删改查数据的标准做法
- **关键特点**：资源导向（URL 代表资源）、无状态、用 HTTP 方法表达操作

### 7. Markdown [文档格式]

- **定义**：一种轻量级标记语言，用纯文本符号（`#`、`*`、`[]()` 等）来排版
- **通俗理解**：比 Word 简单但比纯文本好看的文档格式，程序员最爱用
- **关键特点**：可读性强、易转换为 HTML、几乎所有编程平台都支持

### 8. npm registry / PyPI / GitHub Releases / Maven Central [软件版本源]

- **npm registry**：Node.js/JavaScript 生态的官方软件包仓库（查 Node 相关工具的版本）
- **PyPI**：Python Package Index，Python 官方软件包仓库（查 Python 工具的版本）
- **GitHub Releases**：GitHub 上的官方发布页面（查 VS Code、IDEA 等大型软件的版本）
- **Maven Central**：Java 生态的依赖仓库（查 Maven、JDK 等 Java 工具版本）

### 9. PowerShell / Bash [脚本执行环境]

- **PowerShell**：Windows 系统的高级命令行脚本环境
- **Bash**：macOS/Linux 系统的默认命令行脚本环境
- 提示词要求：生成的安装脚本要适配对应系统

### 10. OpenAI 兼容格式 + DeepSeek API [AI 配置]

- **AI API Key**：调用 AI 服务的密钥（相当于账号密码）
- **Base URL**：AI 服务的接口地址
- **OpenAI 兼容格式**：接口设计遵循 OpenAI 的 Chat Completions API 规范（`/v1/chat/completions`），这样可以无缝切换不同 AI 服务商
- **DeepSeek**：国产大模型服务商，API 格式兼容 OpenAI

### 11. 环境变量 (.env) [配置方式]

- **定义**：操作系统的变量，程序通过它们读取敏感配置信息
- **通俗理解**：把密码、密钥等敏感信息写在 `.env` 文件里，不写死在代码中，更安全

### 12. nvm [预置软件之一]

- **定义**：Node Version Manager，Node.js 版本管理器
- **功能**：一台电脑上安装和切换不同版本的 Node.js

### 13. Homebrew [预置软件之一]

- **定义**：macOS 系统上最流行的包管理器（Package Manager）
- **功能**：通过命令行一键安装各种软件（类似手机上的"应用商店"）

## 五、版本获取策略（三层降级机制）

这是核心逻辑中最复杂的部分，采用三层降级策略：

```
用户搜索软件
    ↓
第一层：查服务端缓存（已经有人查过 → 直接返回）
    ↓ 没缓存
第二层：从软件官方源获取（npm/PyPI/GitHub/Maven 等）
    ↓ 也查不到
第三层：让 AI 提供版本列表（兜底方案）
```

## 六、AI 生成安装方案的差异化策略

提示词做了两个重要区分：

| 软件类型 | 策略 | 例子 |
|----------|------|------|
| **命令行/开源工具** | AI 生成脚本 + 一键安装 | MySQL, Node.js, Docker, Redis |
| **有官方下载页的桌面软件** | 直接给官方链接，不生成脚本 | VS Code, IntelliJ IDEA |

另外明确了 AI 生成时的**禁止事项**：

- 禁止推荐学习资源
- 禁止对软件做多余介绍
- **专注安装本身**

## 七、自主开发循环（AGENT 行为约束）

这是给 AI 代理的行为指令，有四个关键机制：

1. **状态追踪（PROGRESS.md）**：一个项目进度记录文件，让 AI 和人都能随时了解开发到哪了
2. **开发-验证闭环**：写完一个模块 → 立即编译运行验证 → 有错就修 → 通过后再写下一个
3. **端到端验证**：全部写完后，以用户身份完整走一遍流程
4. **防死循环**：同一个 bug 修 5 次还不行，记录后跳过，继续推进其他任务（防止 AI 在一个问题上无限卡死）

## 八、整体数据流

```
用户桌面 APP (Electron + React)
    │
    │ 1. 搜索软件 → 服务端返回版本列表
    │ 2. 选版本 → 服务端调用 AI 生成方案
    │ 3. 查看 Markdown 方案 / 一键安装
    │ 4. 提交反馈（有效/无效）
    │
    ▼
服务端 (FastAPI + SQLite)
    │
    │ 调用 DeepSeek / OpenAI 兼容 API
    │ 存储方案、反馈、统计数据
    │
    ├──→ 管理后台 (React Web)
    │    管理员：管理方案 + 处理反馈
    │
    └──→ AI 服务 (DeepSeek / OpenAI 兼容)
```
