## 角色

你是一个全栈工程师，擅长 Python + Electron + React 开发。



## 任务

开发一个叫「装了吗」的完整系统，帮学编程的同学一键安装软件。包含三个部分：

1. Electron 桌面 APP（用户端）：用户输入想装的软件，选版本和平台，AI 生成 Markdown 安装方案，用户可以一键复制脚本，也可以直接点击「一键安装」由 APP 自动执行脚本完成安装。兼容 Mac 和 Windows
2. Python 服务端：统一调用 AI 生成方案、存储数据、提供 API
3. Web 管理后台：给管理员管理所有方案和用户反馈



## 核心功能

### 桌面 APP（用户端）

1. 用户输入软件名称或一句话（如"我想装 MySQL"），AI 提炼出软件名称，展示版本列表供选择（默认最新稳定版），支持 Windows / macOS / Linux（默认检测当前系统）
2. 版本获取策略：先查服务端缓存 → 再按软件类型从对应源获取（npm registry、PyPI、GitHub Releases、Maven Central 等）→ 都不行则让 AI 提供版本列表
3. AI 生成完整的 Markdown 安装方案：一键安装脚本（含中文注释、环境变量配置、安装验证命令）、脚本运行说明、执行后建议。对于有官方下载页的软件（如 VS Code、IDEA），直接给官方安装链接而非脚本。禁止推荐学习资源，禁止对软件做多余介绍，专注安装本身
4. Markdown 渲染展示，代码块有一键复制按钮
5. 「一键安装」按钮：APP 直接在本地执行安装脚本（Windows 用 PowerShell，macOS 用 Bash），实时显示执行日志，处理权限提升等问题
6. 用户可标记方案「有效」或「无效」，可提交文字反馈
7. 首次启动预置常见软件：MySQL, Node.js, Python, Java (JDK), Git, Maven, Redis, Docker, Nginx, Go, Rust, MongoDB, PostgreSQL, VS Code, IntelliJ IDEA, Homebrew, nvm 等



### 服务端

1. FastAPI + SQLite，提供 RESTful API
2. 统一配置 AI API Key（OpenAI 兼容格式，通过环境变量配置）
3. 存储所有软件信息、安装方案（按 软件+版本+平台 唯一）、用户反馈、选择次数统计



### Web 管理后台

1. 简单密码验证登录
2. 管理已生成方案：查看 / 编辑 Markdown 内容 / 删除 / 让 AI 重新生成
3. 管理用户反馈：查看、标记已处理
4. 查看软件、版本、平台的选择次数统计



## 技术栈

- 桌面框架：Electron
- 前端：React + Shadcn UI，使用 frontend-design 技能美化界面
- 服务端：Python FastAPI + SQLite
- Markdown 渲染：支持代码高亮 + 复制按钮
- Windows 安装脚本用 PowerShell，macOS 用 Bash



## 要求

1. 产品调性轻松搞笑，口号「今天你装了吗？」，UI 活泼有趣
2. 必须生成完整可运行的代码
3. 生成的安装脚本要靠谱：包含中文注释、环境变量配置、安装验证命令、常见问题处理
4. 服务端 AI 配置：读取 Cursor 的 `settings.json` 文件（Windows: `%APPDATA%\Cursor\User\settings.json`，macOS: `~/Library/Application Support/Cursor/User/settings.json`），从中提取 DeepSeek 的 API Key 和 Base URL，写入服务端的 `.env` 文件中作为 `AI_API_KEY` 和 `AI_BASE_URL`。找不到 DeepSeek 配置则降级查找其他 OpenAI 兼容模型的配置（如 openai.apiKey）。接口格式兼容 OpenAI Chat Completions API（/v1/chat/completions）。开发完成后自主测试 AI 生成方案的完整流程，无法自动获取时提示用户手动在 .env 中配置



## 自主开发循环

全程自主开发，不要停下来等我确认，除非遇到无法自行解决的阻塞问题。遵循以下循环机制：

1. 状态追踪：在项目根目录维护 PROGRESS.md，记录当前阶段、已完成事项、进行中事项、问题及解决方案。每完成一个模块就更新
2. 开发-验证闭环：每完成一个模块，立即编译运行验证，有报错就修复，通过后再推进下一个模块
3. 端到端验证：全部功能完成后，以用户视角做一次完整的端到端测试（搜索软件 → 选版本 → 生成方案 → 查看方案），发现问题自行修复
4. 防死循环：同一个问题修复超过 5 次仍未解决，记录到 PROGRESS.md 后跳过，继续推进其他任务
